白话 Skill

第二讲：Skill vs Prompt —— 从"教"到"指挥"的跃迁

核心区别

Prompt是你每次都要教的"口头禅"，是给LLM的"一句话指令"，一次性、临时的。

Skill是AI学会的"职业技能"，是给Agent的"作战方案"，永久的、可重复的。

前者像"你今天帮我做一下这个"，后者像"你以后就知道怎么做了"。

一个形象的类比

Prompt = 给实习生的口头指导
每天说："小张，帮我整理文件，先分类，再排序，然后标出重点，最后存到桌面..."

第二天，又要说一遍...

第三天，同样的话再说一遍...
Skill = 给正式员工的 SOP
入职时学习一次流程文档，以后每天只需说："做一下站会纪要。"

员工学会了流程，不需要每天重复教。

不是替代，是包含

Skill里包含了Prompt，但Prompt不是Skill。

  • 一个Skill可以有多个Prompt（不同的LLM调用步骤）
  • 一个Skill还可以有Agent动作（读文件、执行命令、调用API）
  • 一个Skill还可以有约束规则（禁止做什么、需要审批什么）
  • 一个Skill还可以有错误处理（重试、降级、报警）
Skill = Prompt₁ + Prompt₂ + ... + Agent动作 + 约束规则 + 错误处理 + 资源文件

何时使用？

适合 Prompt
任务简单、不重复、不需要外部工具、不需要多次LLM调用
适合 Skill
重复性 + 信息充分性 + 稳定性 + 可拆分性，四个维度都满足（RISD框架）

何时升级到 Skill？

  • 同一个Prompt粘贴了第3次
  • 每次都要微调格式
  • 团队其他人也在做同样的事
  • 想把这个流程自动化

Skill的本质不是"更复杂的Prompt"，而是"让AI记住怎么做事"。

从Prompt到Skill，是从"临时工"到"正式员工"的转变。

对比表

维度 Prompt Skill
性质 一次性指令 可复用技能
保存方式 临时记忆 永久存储
复杂度 简单 结构化、多步骤
工具调用 有限 完整能力
适用场景 一次性任务 重复性工作
