白话 Skill

第一讲：什么是 Skills？

什么是 Skills？

Skills 是一套指令集，用以指挥 Agent 与 LLM 达成特定的目的。

  • 不是零散的指令，而是一套完整的操作流程
  • 下达对象是执行者，不仅仅是思考者
  • 为解决某一类具体问题而设计

从一个真实场景谈起

Before：每天 200 字的详细指令
"扮演会议纪要专家，提取参会人员、昨日完成、今日计划、阻塞项，用 Markdown 格式输出..."
After：只需一句话
"整理站会纪要。"

AI 自动完成所有步骤，保存文件。

发生了什么变化？不是 AI 变聪明了，而是你给了它一份"作业说明书"。

一个类比

Prompt = 给实习生的口头指导
每天重复，实习生每次按当天理解执行，结果可能偏差。离职后新来的又要从头教起。
Skill = 给员工的 SOP
入职时学习 SOP，以后每天只需说："做一下站会纪要。"员工学会了流程，换人了 SOP 还在。

Skill 长什么样？

一个 Skill 通常是结构化的文件（如 YAML 格式）：

id: daily-standup-summarizer name: 每日站会纪要整理 version: 1.0.0 inputs: - name: transcript type: string required: true workflow: - id: extract_attendees actor: llm prompt: "提取所有参会人员" - id: extract_yesterday actor: llm prompt: "提取每个人的昨日完成" constraints: forbid: - send_email - delete_files

Skill 的四个核心部分

Inputs（输入）
定义用户需要提供什么
Workflow（工作流）
定义核心执行步骤
Constraints（约束）
明确不能做什么
Output（输出）
定义标准化的输出格式
