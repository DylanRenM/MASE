04 / 35 management-principles
开发守则

工程规则

01
设计方案先行
修改代码之前必做设计方案，杜绝拍脑袋编码
02
需求澄清确认
原型确认 + 用例对齐后才进入开发
03
根因分析
任何改错必须先找到根本原因，不碰运气
04
系统化解决
杜绝临时补丁，只做系统性根治方案
05
TDD 驱动
每个功能必须有自动化测试，先写测试再写代码
06
提交节奏
每 20 次对话执行一次 Git 提交
07
备份机制
删除或回退代码前必须做备份
08
合规检查
每次评审必须对照 spec + design 文档
05 / 35 engineering-rules
质量保障

Bug-Fixer 八大原则

01
根因分析
先定位根本原因，再动手修复
02
系统化方案
A/B 双修，杜绝临时补丁
03
回归验证
修复后全量回归测试通过
04
测试反哺
反思：为什么测试没发现？补齐缺口
05
横向扫描
排查同类 BUG，一并修复
06
知识沉淀
记录 lessons + cases，避免重犯
07
三不放过
未复现不放过 · 未归因不放过 · 未沉淀不放过
08
运行时调试
集成 TRAE-debugger，HTTP 日志 + 科学调试
