AI 编程时代的新命题

随着 AI 编程工具的普及，"如何管理 AI"已经成为软件工程领域的新命题。

OSSP → PDP → Harness 的三层体系系统化管理人机协作过程。

核心问题

你和你的团队，现在的 Harness 到了什么水平？还差多远才能达到 "人马合一"？

借鉴 CMMI 成熟度模型思想，设计 Harness 成熟度模型（HMM），帮助量化评估、诊断差距、规划改进路径。

L1 & L2 详解
05 / 12

L1 初始级：个人英雄主义时代

一句话概括：Harness 不存在，全靠个人悟性。

典型表现：零散的个人习惯，每次从零开始写 Prompt，踩坑不留痕。

痛点：同一个 Bug 反复出现，新人无指南可参考，AI 输出质量不稳定。

L2 可管理级：项目级秩序初现

一句话概括：项目级规则开始建立，基础复用成为可能。

典型表现：有 project_rules.md（3-10 条核心规则），1-3 个基础 Skill，有 bug-fixes.md 记录历史 Bug。

核心能力：同一个坑基本不踩第二次，团队有一致的 AI 使用底线规则。

L3 已定义级
06 / 12

L3 已定义级：组织级标准确立

一句话概括：组织级 Harness 框架建立，三级体系（组织→项目→个人）完整贯通。

组织级 Harness 框架结构

├── policy/policy.md        ← 组织方针
├── ossp/ossp.md            ← 标准软件流程
├── lcm/lcm.md              ← 生命周期定义
├── process/process.md      ← 过程定义
├── templates/              ← 标准模板
├── guidelines/             ← 最佳实践指南
├── checklists/             ← 检查单
├── dod/                    ← 完工准则
└── lessons-learned/        ← 经验教训库

核心能力：新项目直接复制组织级模板快速建立 Harness，不同项目可选择重装/轻装模式，经验教训跨项目共享。

L5 优化级：AI 自驱动演进

一句话概括：AI 自动提炼经验、建议规则、预判风险。Harness 成为自我进化的有机体。

智能系统架构

经验教训库 → AI 分析引擎 → 自动提炼规则 → 规则库更新

新项目启动 → AI 评估引擎 → 推荐裁剪方案

开发者使用 → AI 学习引擎 → 个人 Harness 自动优化

核心能力示例

  • 自动规则提炼：从历史 Bug 中自动发现模式
  • 智能裁剪推荐：根据项目特征自动推荐方案
  • 个人适配优化：学习开发者偏好自动调整
  • 预测性质量：预判可能出现的问题模式

自评问卷

对以下每项评估，得分 1-5（对应 L1-L5），最后取平均值：

  1. 组织级 Harness 是否存在并被维护？
  2. 项目级规则是否体系化、可裁剪？
  3. 个人 Harness 是否可与项目/组织级同步？
  4. 经验是否被系统化记录和复用？
  5. 是否有度量指标来衡量 Harness 效能？
  6. 重装/轻装模式是否可以灵活切换？
  7. Skill 库是否完善且被持续使用？
  8. Bug 修复后是否触发规则/Skill 改进？

评分解释

1.0-1.9L1 初始级开始记录 bug-fixes.md
2.0-2.9L2 可管理级建立组织级模板
3.0-3.9L3 已定义级开始收集度量数据
4.0-4.9L4 量化管理级探索 AI 自动分析

升级路径指南
11 / 12

L1 → L2：从个人到项目

  • 创建 docs/bug-fixes.md，记录每一个 Bug
  • 从 Bug 记录中提炼 3-5 条项目规则
  • 引入 bug-fixer Skill，建立固定修复流程

预计时间：2-4 周

L2 → L3：从项目到组织

  • 提取项目 Harness 的通用部分为组织级模板
  • 建立 Policy、OSSP、LCM、Process 等资产
  • 完善 Templates、Guidelines、Checklists、DoD

预计时间：1-3 个月

L3 → L4 → L5

L3→L4：定义效能指标体系，建立数据采集机制，定期生成效能报告。（2-6 个月）

L4→L5：建立经验向量化存储，开发规则自动提炼的 AI Pipeline，构建组织级知识图谱。（6-12 个月）

总结
12 / 12

Harness 成熟度模型核心要点

  • 成熟度不是目的，是手段 — 不是每个组织都需要达到 L5，适合的才是最好的
  • 层级递进，不可跳级 — 没经过 L2 的项目级积累，直接搞 L3 会水土不服
  • 度量的价值在于行动 — L4 的数据如果不转化为规则改进，就只是数字
  • L5 是人机协作的理想状态 — AI 帮你管理 AI，你只需关注创造性决策
