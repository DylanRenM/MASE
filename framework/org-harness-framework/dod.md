# 完工准则（DOD - Definition of Done）

## 1. DOD 概述

本文件定义了组织级的完工准则，用于判断各项活动和交付物是否完成。

## 2. 需求规格说明 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 完整性 | 所有需求都已识别并记录 |
| 清晰性 | 需求描述清晰、无歧义 |
| 可测试性 | 每个需求都有明确的验收标准 |
| 一致性 | 需求之间无冲突 |
| 可行性 | 需求在技术和资源上可行 |
| 优先级 | 所有需求都已确定优先级 |
| 确认 | 需求已获得利益相关者确认 |
| 评审 | 需求文档已通过评审 |

## 3. 设计文档 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 完整性 | 所有模块的设计都已完成 |
| 正确性 | 设计满足所有需求 |
| 一致性 | 设计文档内部一致 |
| 可实现性 | 设计在技术上可实现 |
| 安全性 | 安全考虑已纳入设计 |
| 性能 | 性能考虑已纳入设计 |
| 评审 | 设计文档已通过技术评审 |
| 文档 | 设计文档完整且可理解 |

## 4. 代码 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 功能 | 代码实现了所有需求 |
| 编译 | 代码通过编译 |
| 测试 | 代码通过所有单元测试 |
| 评审 | 代码已通过代码评审 |
| 规范 | 代码符合编码标准 |
| 注释 | 代码有适当的注释 |
| 文档 | 相关文档已更新 |
| 安全性 | 代码无安全漏洞 |

## 5. AI 生成代码 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 生成 | AI 已生成代码初稿 |
| 评审 | 代码已通过人工评审 |
| 修改 | 评审发现的问题已修复 |
| 测试 | 代码通过所有测试 |
| 验证 | 代码功能已验证 |
| 文档 | 代码文档已补充 |
| 记录 | AI 使用记录已保存 |

## 6. 测试 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 覆盖 | 所有需求都有测试覆盖 |
| 单元测试 | 单元测试覆盖率达到目标 |
| 集成测试 | 集成测试通过 |
| 系统测试 | 系统测试通过 |
| 验收测试 | 验收测试通过 |
| 缺陷 | 所有严重缺陷已修复 |
| 报告 | 测试报告完整 |
| 文档 | 测试文档已归档 |

## 7. 部署 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 环境 | 部署环境已准备就绪 |
| 配置 | 配置文件已正确配置 |
| 迁移 | 数据库迁移已完成 |
| 验证 | 部署后验证已通过 |
| 文档 | 部署文档已更新 |
| 培训 | 用户培训已完成 |
| 回滚 | 回滚计划已制定 |
| 监控 | 监控已配置 |

## 8. 项目 DOD

| 检查项 | 完成标准 |
|--------|---------|
| 交付 | 所有功能已交付 |
| 质量 | 所有测试已通过 |
| 文档 | 所有文档已归档 |
| 知识 | 知识转移已完成 |
| 经验 | 经验总结已完成 |
| 验收 | 项目已通过验收 |
| 资源 | 资源已释放 |
| 关闭 | 项目已正式关闭 |
