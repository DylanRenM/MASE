## Why

用户拥有上百本八字命理电子书，这些书内容交叉重复严重，但各家观点和命例又各具互补价值。手动对比阅读极为低效，无法系统性地获取某一流派在某一专题上的完整知识。将这批书籍自动融合为一本结构化的百科全书，可以消除重复、标注分歧、汇集各家之长，大幅提升知识获取效率。

## What Changes

- 新增**电子书批量解析管道**：支持 EPUB/PDF/TXT/扫描件格式，自动提取文本并分块
- 新增**本地向量化与分类系统**：使用本地 Embedding 模型 + Qdrant 向量数据库，对内容做语义索引和流派/专题初步分类
- 新增**聚类去重管道**：DBSCAN 向量聚类 + 本地千问 LLM 精判，识别重复/互补/冲突内容
- 新增**百科全书编排引擎**：按 [流派×专题] 二维结构组织知识条目，融合互补内容，标注流派分歧
- 新增**命例数据库**：按专题分类，支持多维度筛选和与百科互跳
- 新增**Web 管理界面**：左栏书籍管理（增量添加、状态追踪）+ 右栏百科浏览（树状目录 + 内容渲染）

## Capabilities

### New Capabilities

- `book-parser`: 电子书解析与分块，支持 EPUB/PDF/TXT/扫描OCR
- `content-vectorizer`: 内容向量化、标签分类与语义索引
- `dedup-pipeline`: 聚类去重管道（向量粗筛 + LLM 精判）
- `encyclopedia-builder`: 百科条目编排与命例数据库生成
- `web-ui`: 双面板 Web 管理界面（书籍管理 + 百科浏览）

### Modified Capabilities

_（全新项目，无现有能力需修改）_

## Impact

- 需要本地部署：Qdrant 向量数据库、bge-large-zh-v1.5 Embedding 模型、Qwen3-32B LLM
- 存储：约 150MB 向量 + ~500MB 文本元数据 + JSON 百科内容
- 处理时间：100本书全量处理约 6-8 小时（本地 GPU 推理）
- 无外部 API 依赖，无网络要求
